pytest tests/ -v

## 엣지 케이스 공통 원칙

1. **빈 입력** — 모든 모듈은 빈 문자열을 받아도 예외 없이 기본값 반환
2. **None 입력** — `Optional` 인자는 None 처리 필요
3. **인코딩** — 한글, 특수문자 포함 텍스트에서 깨짐 없이 동작
4. **긴 텍스트** — 1000자 이상 텍스트에서 성능 이슈 없음
5. **날짜 오인식** — 날짜처럼 보이지만 날짜가 아닌 숫자 (전화번호 등)에서 오파싱 금지

## 테스트 작성 가이드

```python
# 예시 구조
def test_케이스명():
    # Arrange
    input_text = "..."
    # Act
    result = 함수(input_text)
    # Assert
    assert result == 기대값
```

- 테스트명은 `test_<동작>_<조건>` 형식 권장
  - 예: `test_extract_deadline_absolute_date`
- `assert` 실패 메시지 포함 권장: `assert result == x, f"got {result}"`
